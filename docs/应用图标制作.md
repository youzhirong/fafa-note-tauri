# 制作 / 更换应用图标

记录用自己的图片生成 fafa-note 全套应用图标的完整步骤（含踩过的坑）。

## 关键前提

- **源图必须是正方形**，否则 `tauri icon` 直接报错：
  `Error Source image must be square`
- 建议源图 **≥ 1024×1024**、PNG 格式。背景**填充色**比透明更适合做图标
  （透明四角在 macOS/Windows 上反而显小、发怪）。

## 步骤

### 1. 把图片做成正方形（用 macOS 自带的 sips）

假设原图在 `public/icon.png`（非正方形）：

```bash
cd /Users/youzhirong/Documents/AikoProject/fafa-note-tauri

# 方式①（最简单）：直接缩放为 1024×1024。非正方形会被轻微拉伸，
# 差几个像素时形变 <1%，肉眼基本看不出。
sips -z 1024 1024 public/icon.png --out /tmp/icon-square.png

# 方式②（不形变）：按最长边补成正方形画布，居中。padColor 用十六进制(无#)，
# 尽量接近原图背景色，例如奶油色 FBF4E6；不确定就用 ffffff。
# 先看尺寸： sips -g pixelWidth -g pixelHeight public/icon.png
sips --padToHeightWidth 1344 1344 --padColor FBF4E6 public/icon.png --out /tmp/icon-square.png
```

### 2. 生成全套图标

```bash
npm run tauri icon /tmp/icon-square.png
```

会在 `src-tauri/icons/` 生成并覆盖：
`32x32.png`、`128x128.png`、`128x128@2x.png`、`icon.icns`(macOS)、`icon.ico`(Windows)
以及 iOS/Android 的整套图标。`tauri.conf.json` 已引用这些文件，无需改配置。

### 3. 重新打包，让图标生效

```bash
npm run tauri:build
```

产物：`src-tauri/target/release/bundle/dmg/*.dmg`（macOS）。
Windows/Linux 的图标会在 CI（GitHub Actions）打包时一并使用，见 auto-update-github.md。

## 常见错误

| 报错 | 原因 / 解决 |
|------|-------------|
| `Source image must be square` | 源图不是正方形 → 用上面的 sips 命令处理 |
| `Operation not permitted` 读文件 | macOS 隐私(TCC)：运行程序无权读某文件夹 → 给「终端」开**完全磁盘访问权限**，或把图片放到 `/tmp` 再处理 |
| 换了图标但应用里还是旧图标 | 没重新 `tauri:build`；或系统图标缓存，重启 Dock：`killall Dock` |

## 备注：把图抠成透明（可选）

macOS「预览/照片」可一键「抹除背景 / 拷贝主体」，导出透明 PNG；
或用 Python 的 `rembg`（`pip install rembg`，AI 抠图、图片不外传）。
抠好后同样先确保正方形，再走上面步骤。
