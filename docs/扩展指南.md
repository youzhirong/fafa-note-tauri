# 扩展指南

本文用「照着做」的方式说明常见扩展任务。

---

## 一、新增一个菜单栏项 + 页面（最常见）

例：加一个「回收站」功能。

### 1. 新建页面组件 `src/views/TrashView.vue`

```vue
<script setup lang="ts">
import { useNotesStore } from '@/stores/notes'
const store = useNotesStore()
</script>

<template>
  <div class="fafa-scroll" style="padding: 24px">
    <h2>回收站</h2>
    <!-- 你的内容 -->
  </div>
</template>
```

### 2. 注册路由 `src/router/index.ts`

在 `routes` 数组里加一条：

```ts
{
  path: '/trash',
  name: 'trash',
  component: () => import('@/views/TrashView.vue'),
  meta: { title: '回收站' },
},
```

### 3. 加入左侧菜单 `src/config/navigation.ts`

在 `NAV_ITEMS` 里加一项（`route` 要和上面路由的 `path` 一致）：

```ts
{ key: 'trash', label: '回收站', icon: 'pi pi-trash', route: '/trash', placement: 'top' },
```

完成。菜单栏会自动出现「回收站」，点击跳转。图标用
[PrimeIcons](https://primevue.org/icons) 的 class，如 `pi pi-star`。

> `placement: 'bottom'` 可把项目固定到菜单栏底部（如「设置」）。

## 二、给笔记增加一个字段

例：给笔记加「颜色标记」`color`。

1. **类型** `src/types/index.ts`：在 `Note` 接口加 `color?: string`。
2. **存储迁移** `src/services/repository/IndexedDbNoteRepository.ts`：
   如果只是新增可选字段，旧数据读出来是 `undefined`，通常无需迁移；
   若要建索引或填默认值，新增一个 Dexie 版本：

   ```ts
   this.version(2).stores({
     notes: 'id, folderId, updatedAt, pinned, title, color',
   }).upgrade(async (tx) => {
     await tx.table('notes').toCollection().modify((n) => { n.color = n.color ?? '' })
   })
   ```

   ⚠️ 不要修改已发布的 `version(1)` 声明，否则用户本地数据迁移会失败。
3. **UI**：在 NoteEditor / NoteList 里读写 `note.color`。

## 三、把桌面端存储换成 SQLite（性能优化）

数据量很大时，桌面端可用 SQLite + FTS5 获得更快的原生全文搜索。
Web 端仍用 IndexedDB（浏览器跑不了 SQLite）。

1. 安装：`npm i @tauri-apps/plugin-sql`
2. `src-tauri/Cargo.toml` 加依赖：
   ```toml
   tauri-plugin-sql = { version = "2", features = ["sqlite"] }
   ```
3. `src-tauri/src/lib.rs` 注册插件：
   ```rust
   .plugin(tauri_plugin_sql::Builder::default().build())
   ```
4. `src-tauri/capabilities/default.json` 增加 sql 权限。
5. 参照 `src/storage/sqlite/SqliteNoteRepository.ts.example` 实现
   `INoteRepository`（去掉 `.example` 后缀）。
6. 在 `src/services/repository/index.ts` 工厂里按平台返回：
   ```ts
   import { isTauri } from '@/platform'
   import { SqliteNoteRepository } from '@/storage/sqlite/SqliteNoteRepository'
   import { IndexedDbNoteRepository } from './IndexedDbNoteRepository'

   instance = isTauri() ? new SqliteNoteRepository() : new IndexedDbNoteRepository()
   ```

上层（store / UI）完全不用改 —— 这就是 Repository 抽象的价值。

## 五、调用第三方接口（注意跨域）

统一用 `src/services/HttpService.ts` 的 `httpRequest` / `httpGetJson`：

```ts
import { httpGetJson } from '@/services/HttpService'
import { useSettingsStore } from '@/stores/settings'

const settings = useSettingsStore()
```

- Web 端会自动套用户在「设置」里配置的代理地址绕过 CORS；
- Tauri 端直连，无需代理。

## 关于 Markdown 预览渲染

- 三种视图模式在 `NoteEditor.vue`：`edit`（仅编辑，`MdEditor :preview="false"`）、
  `live`（编辑+预览）、`preview`（纯预览，用独立的 `MdPreview` 组件，渲染样式来自
  已导入的 `md-editor-v3/lib/style.css`，包含 `.md-editor-preview` 样式）。
- **代码块语法高亮已离线化**：见 `src/config/mdEditorSetup.ts`——用本地 highlight.js 实例
  注入 md-editor 的 `config()`，并引入本地高亮主题 CSS，不再依赖 CDN。该文件在
  `NoteEditor.vue` 里以副作用方式 import，使 highlight.js 随「笔记页」懒加载、不拖累首屏。
  （mermaid / katex 仍走 CDN，如需离线可同法接本地库。需要暗色高亮主题时把
  `github.css` 换成 `github-dark.css`。）
- **`preview` prop 仅初始化生效**：md-editor-v3 的 `preview` 只在挂载时读取一次，运行中改它
  不会切换预览栏。本项目的「编辑 / 编辑预览」切换靠给 `<MdEditor :key="viewMode">` 强制重挂载实现。
- **纯预览的两处库 CSS 坑（已在 `NoteEditor.vue` 的 `<style>` 修正，勿删）**：
  1. *正文贴边*：v5 的 `MdPreview` 根节点带 `.md-editor-previewOnly`，库里把
     `.md-editor-previewOnly .md-editor-preview` 的 `padding` 重置为 `0`，导致纯预览正文左右贴边，
     滚动条（在外层 `.md-preview`，`overflow-y:auto`）还会压住右侧文字。修正：
     `.md-preview :deep(.md-editor-preview){ padding:16px 20px 48px }` 补回内边距。
     （注意：v5 已无 `.md-editor-preview-wrapper` 可用内边距挂点，旧写法是死代码。）
  2. *代码头盖住弹窗*：代码块头部 `.md-editor-code-head` 库内默认 `position:sticky; z-index:10000`，
     会盖住 PrimeVue 的 modal 弹窗（如 NoteList 的「新建文件夹」，默认 z-index ~1100），表现为
     纯预览下新建分类弹窗被代码语言名/复制按钮那一条遮挡。修正：
     `.editor-body :deep(.md-editor-code-head){ z-index:2 }`，吸顶效果不变又不会高过任何对话框。

## 双链 [[标题]] 的实现

- 渲染：`src/config/mdEditorSetup.ts` 里给 md-editor 的 `markdownItConfig` 注册了一个
  markdown-it 行内规则，把 `[[标题]]` 输出成 `<a class="wikilink" data-wikilink="标题">`。
- 点击：`NoteEditor.vue` 在编辑区容器上做事件委托，命中 `.wikilink` 时调
  `store.openOrCreateByTitle(title)`（同名笔记则打开，否则新建并打开）。
- 反向链接：`NoteEditor.vue` 的 `backlinks` 计算属性扫描所有未删除笔记，找出正文里
  含 `[[当前标题]]` 的笔记并列在编辑器底部。标题匹配为大小写不敏感、去空白。

## 启用桌面端自动更新（关于页「检查更新」）

代码已接好 `@tauri-apps/plugin-updater` + `plugin-process`（仅桌面端注册），
关于页的「检查更新」会调用它。但要真正能更新，还需三步配置（否则点击会提示「更新服务未配置」）：

1. **生成签名密钥**：`npm run tauri signer generate -- -w ~/.tauri/fafa-note.key`
   会得到一对公钥/私钥。**私钥务必保密**（用于给发布包签名），公钥填进配置。
2. **配置 tauri.conf.json**，加入 updater：
   ```json
   "plugins": {
     "updater": {
       "endpoints": ["https://你的域名/updates/{{target}}-{{arch}}/{{current_version}}"],
       "pubkey": "粘贴上一步的公钥"
     }
   }
   ```
3. **发布**：每次 `tauri build` 用私钥签名（设环境变量 `TAURI_SIGNING_PRIVATE_KEY`），
   把生成的安装包和 `latest.json`（更新清单）上传到上面的 endpoint。
   清单格式见 Tauri 官方文档「Updater」。

Web 端无此逻辑——重新构建托管即最新（关于页对 Web 显示「刷新即最新」）。

## ⚠️ Tauri 注意事项（避坑）

- **`window.prompt` / `alert` / `confirm` 在 Tauri 的 WebView 里不可用**（`prompt` 直接返回
  `null`），所以不要用它们做交互。需要输入/确认时用 PrimeVue 的 `Dialog` / `useConfirm`
  （本项目新建文件夹就是用 `Dialog` 实现的，见 `NoteList.vue`）。
- 调用任何 Tauri 插件前用 `isTauri()` 守卫，并用动态 `import`，避免影响纯 Web 构建。
