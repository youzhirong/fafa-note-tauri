# 打包与构建安装程序

## 产物输出位置

所有安装包都在：

```
src-tauri/target/release/bundle/
├─ dmg/        macOS 安装镜像 (*.dmg)
├─ macos/      macOS 应用 (*.app)
├─ nsis/       Windows 安装程序 (*-setup.exe)
├─ msi/        Windows MSI 安装包 (*.msi)
├─ deb/ appimage/ rpm/   Linux 包
```

---

## 一、在 Mac 上打包 macOS 安装包（原生，推荐）

```bash
npm install            # 首次或依赖有更新时
npm run tauri:build
```

完成后到 `src-tauri/target/release/bundle/dmg/` 拿 `.dmg`，
或 `bundle/macos/` 拿 `.app`。

> 默认只构建**当前机器架构**。你这台是 Intel(x86_64)，产出的就是 Intel 包。

### 想同时支持 Intel + Apple Silicon（通用二进制）

```bash
rustup target add aarch64-apple-darwin x86_64-apple-darwin
npm run tauri build -- --target universal-apple-darwin
```

## 二、Windows 的 .exe —— ⚠️ 不能在 Mac 上直接交叉编译

Tauri **官方不支持跨平台编译**：Windows 端依赖 WebView2 运行时和 NSIS / WiX（MSVC 工具链），
必须在 **Windows 环境**里构建。在 Mac 上要拿到 Windows 安装包，有三条路：

### 方案 A（最推荐）：用 GitHub Actions 云端构建

本项目已自带 `.github/workflows/release.yml`。把代码推到 GitHub 后，打一个版本标签：

```bash
git tag v0.1.0
git push origin v0.1.0
```

工作流会在 **windows-latest** 上原生构建出 `*-setup.exe`(NSIS) 和 `.msi`，
同时还构建 macOS / Linux 包，全部发布到对应 GitHub Release。**你完全不需要本地 Windows。**
（这套也顺带支撑应用内自动更新，详见 [auto-update-github.md](auto-update-github.md)。）

### 方案 B：用 Windows 机器 / 虚拟机

在 Windows（真机，或 Mac 上的 Parallels / UTM 虚拟机）里装好：
- Node.js
- Rust（含 MSVC：安装时选 “Desktop development with C++” / Build Tools）
- WebView2 Runtime（Win10/11 一般已自带）

然后：

```bash
npm install
npm run tauri:build
# 产物：src-tauri/target/release/bundle/nsis/*-setup.exe
```

### 方案 C（进阶，不推荐生产用）：cargo-xwin 交叉编译

社区方案，可在 macOS/Linux 上交叉编译 Windows 包，但坑较多（WebView2、签名、NSIS 资源处理），
**不保证稳定、无法做代码签名**：

```bash
rustup target add x86_64-pc-windows-msvc
cargo install cargo-xwin
npm run tauri build -- --runner cargo-xwin --target x86_64-pc-windows-msvc
```

仅当你能接受折腾、且不需要签名时尝试；正式发布请走方案 A。

## 五、关于签名 / 安全提示

- **macOS**：分发给别人需要 Apple 开发者证书做签名 + 公证(notarize)，否则对方首次打开会被
  Gatekeeper 拦截。
- **Windows**：未签名的安装包会触发 SmartScreen 警告；正式发布建议购买代码签名证书。
- 上述是「分发签名」；自动更新用的「更新包签名」是另一套密钥（`tauri signer`），见 auto-update-github.md。

> ⚠️ 注意区分两种「签名」：
> - **更新包签名**（`TAURI_SIGNING_PRIVATE_KEY` / `tauri.conf.json` 里的 `pubkey`）——只保证自动更新
>   下载的包没被篡改，**和 macOS「已损坏」拦截无关**。本仓库 CI 已配好这套。
> - **Apple 分发签名 + 公证**——决定别人下载后能不能直接双击打开。**本仓库尚未配置**，所以现在
>   下载的 macOS 包首次打开会报「已损坏」（见下面的「临时绕过」与「正式解决」）。

### 5.1 macOS「已损坏，无法打开」——临时绕过（无需证书，免费）

未签名/未公证的包从网上下载后会被打上 `com.apple.quarantine` 隔离属性，Gatekeeper 直接判定为
「已损坏」。让使用者去掉隔离属性即可正常打开：

```bash
# 装到「应用程序」后执行（提示输入开机密码）
sudo xattr -rd com.apple.quarantine /Applications/fafa-note.app
```

> 「已损坏」这种提示**右键 →「打开」一般无效**（右键打开只对「无法验证开发者」有用），必须用 `xattr`。
> 这点已写进 README，方便直接转给下载用户。

### 5.2 正式解决：Apple 签名 + 公证（需 Apple 开发者账号，$99/年）

有了账号后，CI 里 `tauri-action` 原生支持自动签名+公证，下载的包就能直接双击打开、不再报损坏。

**准备（一次性）：**

1. 在 [developer.apple.com](https://developer.apple.com) 注册开发者账号（$99/年）。
2. 在「Certificates」创建 **Developer ID Application** 证书，下载后导出为 `.p12`（设个密码）。
3. 把 `.p12` 转成 base64 备用：
   ```bash
   base64 -i certificate.p12 | pbcopy   # 已复制到剪贴板
   ```
4. 在 [appleid.apple.com](https://appleid.apple.com) 给你的 Apple ID 生成一个 **App 专用密码**（公证用）。
5. 记下你的 **Team ID**（开发者后台 Membership 页可查）和签名身份字符串，形如
   `Developer ID Application: 你的名字 (TEAMID)`。

**在 GitHub 仓库 Settings → Secrets and variables → Actions 添加：**

| Secret 名 | 值 |
|-----------|----|
| `APPLE_CERTIFICATE` | 上面 base64 后的 `.p12` 内容 |
| `APPLE_CERTIFICATE_PASSWORD` | 导出 `.p12` 时设的密码 |
| `APPLE_SIGNING_IDENTITY` | `Developer ID Application: 你的名字 (TEAMID)` |
| `APPLE_ID` | 你的 Apple ID 邮箱 |
| `APPLE_PASSWORD` | 上面生成的 **App 专用密码** |
| `APPLE_TEAM_ID` | 你的 Team ID |

**改 `.github/workflows/release.yml`**：在 `Build & release (Tauri)` 那一步的 `env:` 下追加
（与已有的 `TAURI_SIGNING_*` 并列）：

```yaml
      - name: Build & release (Tauri)
        uses: tauri-apps/tauri-action@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
          TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
          # ↓ 新增：Apple 分发签名 + 公证（仅 macos-latest 任务会用到，其它平台忽略）
          APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
          APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
          APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
          APPLE_ID: ${{ secrets.APPLE_ID }}
          APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
          APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
        with:
          # ……其余 with 配置保持不变……
```

配置后重新打标签发布（`git tag v1.0.4 && git push origin v1.0.4`），CI 产出的 macOS 包就是
已签名公证的，任何人下载双击即可打开，无需再跑 `xattr`。

> Windows 同理，未签名会触发 SmartScreen；要消除需购买代码签名证书并配置
> `WINDOWS_CERTIFICATE` 等 Secret，参考 tauri-action 文档。

## 七、常见崩溃排查

- **安装后打开闪退 / `abort() / SIGABRT`**：多半是 Rust 启动期 panic（release 里 `panic=abort` 会让
  崩溃日志只剩地址）。**最快定位方法**：终端直接运行二进制看 panic 文本——
  ```bash
  ./src-tauri/target/release/fafa-note-tauri
  ```
  典型原因：
  - `PluginInitialization("updater", ... invalid type: null ...)`：注册了 updater 插件但
    `tauri.conf.json` 缺少 `plugins.updater` 配置。要么补全配置（endpoints+pubkey），要么去掉插件注册。
  - `pubkey` 结尾误带 `%`（zsh 复制无换行文件所致），会导致更新校验失败。
- **`createUpdaterArtifacts: true` 但本地构建报「private key not set」**：本地构建需要签名私钥；
  本仓库默认 `false`，发布交给 CI（见 auto-update-github.md）。
